# BottomSheet

## Purpose

`BottomSheet` presents a focused, dismissible surface over the current screen for a contained task or
choice — picking a payment method, confirming a detail, reading a short explainer — without navigating
away. A dimming scrim pulls focus to the sheet; the screen stays visible behind it for context.

## Usage

- Drop it inside the **screen container** (a positioned parent). It renders as an absolute overlay that
  fills that parent, so it stays inside a device frame rather than portalling to the document.
- Gate it with `open`, or conditionally render `<BottomSheet>` yourself.
- Put the confirming action in the `footer` slot (a full-width `Button`); it clears the home indicator.
- The scrim and the corner close button both call `onClose`.

## Do not use

- ❌ For a full-screen task or a multi-step flow — navigate to a screen instead.
- ❌ For transient, non-blocking feedback — use a Toast/Snackbar.
- ❌ As a permanent panel — a sheet is modal and dismissible by design.
- ❌ With raw colours/sizes — every value comes from tokens.

## Variants

Composition props rather than visual variants: `footer` (pinned action), `showClose`, `showGrabber`,
`title`. See **WithoutFooter**, **NoGrabber**, **Scrollable**, and **Interactive**.

## States

Open (shown) · closed (`open={false}` → renders nothing). The body scrolls when content exceeds the
capped height; header, footer and corners stay fixed.

## Accessibility

- The scrim is a button labelled **"Dismiss"**; the corner button is labelled **"Close"** — both fire
  `onClose`, giving keyboard/AT users an explicit exit.
- Footer actions are real `Button`s with their own roles/labels.

## Responsive behavior

Full-bleed width, anchored to the bottom; height caps at **90%** of the host so the screen stays
partly visible. Content scrolls within that cap.

## Platform notes

Renders as an in-parent absolute overlay (not a native `Modal`/portal) so it stays contained in device
frames and previews. No enter/exit animation yet — that lands with drag-to-dismiss gesture support.

## Bottom-sheet rules (tokens)

| Aspect | Token |
| --- | --- |
| Top corners (bottom flush) | `radius.2xl` (24) |
| Scrim | `effects.overlay-default` @ `opacity.40` |
| Surface elevation | `elevation.sheet` |
| Grabber handle | `size.32 × size.4`, `radius.pill` |
| Corner close button | `size.32` circle, `background.secondary`, `space.md` inset |
| Body / footer padding | `space.md` |
| Footer home-indicator inset | `safeArea.bottom` (34) |
| Max height | 90% of host |
| Stacking | `zIndex.modal` |

## Tokens used

`radius.2xl` · `radius.pill` · `effects.overlay-default` · `opacity.40` · `elevation.sheet` ·
`background.primary` · `background.secondary` · `border.default` · `icon.secondary` · `space.md` ·
`space.sm` · `size.32` · `size.48` · `size.4` · `safeArea.bottom` · `zIndex.modal` · `typography.titleSmall`.
